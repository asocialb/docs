# Быстрая генерация URL

## 1. Как быстро получить URL

Если вы хотите сгенерировать набор URL для тестирования, достаточно открыть [Консоль CSS > Доступ к CSS-коду > Генератор Push-ссылок](https://console.tencentcloud.com/live/livecodemanage) и нажать кнопку **Сгенерировать Push URL**, чтобы получить push-ссылку и три ссылки воспроизведения с различными протоколами.

![img](https://staticintl.cloudcachetci.com/cms/backend-cms/b37a0172d44f11eea2b0525400bb593a.png)

## 2. Как реализовать автоматическое формирование URL на серверной стороне

### 2.1 Push URL

В реальном сценарии невозможно вручную создавать push- и playback-ссылки для каждого стримера. Вместо этого эти URL формируются автоматически вашим сервером. Любой URL, соответствующий спецификациям Tencent Cloud, может использоваться как push-ссылка. Стандартный push URL показан ниже и состоит из трёх частей:

![url](https://staticintl.cloudcachetci.com/cms/backend-cms/b36b789ad44f11eea2b0525400bb593a.png)

- **CSS Code**
Также называется идентификатором комнаты (room ID). Рекомендуется использовать случайные числа или идентификатор пользователя. В допустимом CSS-коде необходим префикс BIZID.
- **txTime**
Указывает время истечения срока действия URL. Формат — метка времени UNIX в шестнадцатеричном формате, например, 5867D600 означает, что URL истечёт 1 января 2017 года в 00:00:00. Как правило, txTime устанавливается на 24 часа позже текущего времени. Не рекомендуется устанавливать слишком короткий срок действия, чтобы избежать невозможности восстановления push-потока стримером в случае кратковременного сбоя сети во время трансляции.
- **txSecret**
Это подпись защиты от хотлинкинга, которая используется для предотвращения имитации вашего серверного приложения злоумышленниками для генерации push URL. Подробнее о методе вычисления см. [Вычисление защиты от хотлинкинга](https://intl.cloud.tencent.com/document/product/267/7977).
- **Пример кода**
Перейдите в [Консоль CSS > Доступ к CSS-коду > Генератор Push-ссылок](https://console.tencentcloud.com/live/livecodemanage). В нижней части страницы представлен пример кода (PHP и Java), демонстрирующий генерацию URL с защитой от хотлинкинга.

## 3. Как рассчитать защиту от хотлинкинга

Защита от хотлинкинга — это поле **txSecret** в push- и playback-ссылках. Оно используется для предотвращения имитации злоумышленниками вашего серверного приложения для генерации push URL или взлома вашего playback URL в своих целях.

### 3.1 Принцип работы

Чтобы предотвратить имитацию злоумышленниками вашего сервера для генерации push URL, необходимо настроить в консоли CSS

**ключ шифрования защиты от хотлинкинга**

, который злоумышленники не смогут получить для подделки действительного push URL. На рисунке ниже показан принцип работы.

![img](https://staticintl.cloudcachetci.com/cms/backend-cms/b37188a5d44f11ee9409525400c26da5.png)

### 3.2 Порядок вычисления

- **Шаг 1: Обмен ключом**
Необходимо согласовать **ключ шифрования** в консоли на официальном сайте. Этот ключ шифрования используется для генерации подписи защиты от хотлинкинга на вашем сервере. Поскольку у Tencent Cloud тот же ключ, что и у вас, он может расшифровать и проверить подпись защиты от хотлинкинга, сгенерированную вашим сервером.

Ключи шифрования подразделяются на **ключи защиты от хотлинкинга для push** и **ключи защиты от хотлинкинга для воспроизведения**. Первые используются для генерации push-ссылок с защитой от хотлинкинга, а вторые — для генерации playback-ссылок с защитой от хотлинкинга. Настроить ключи защиты от хотлинкинга для push можно в [Консоли CSS](https://console.tencentcloud.com/live), как показано ниже:

![](https://staticintl.cloudcachetci.com/cms/backend-cms/b36c4fe1d44f11ee9409525400c26da5.png)

> **Защита от хотлинкинга для воспроизведения по умолчанию отключена:** Поскольку конфигурация ключа защиты от хотлинкинга для воспроизведения должна быть синхронизирована с тысячами CDN-кластеров, ключ нельзя часто изменять на этапе отладки из-за длительного периода синхронизации. Если вам необходимо настроить защиту от хотлинкинга для воспроизведения, свяжитесь с нами через службу поддержки. Как правило, синхронизация по всем кластерам занимает от 1 до 3 дней.

- **Шаг 2: Генерация txTime**
В подписи открытым текстом является txTime, указывающий срок действия URL. Например, если текущее время — 2016-07-29 11:13:45, а сгенерированный URL должен истечь через 24 часа, txTime можно установить на 2016-07-30 11:13:45.

Такая длинная строка времени заняла бы слишком много места в URL. В реальном сценарии 2016-07-30 11:13:45 преобразуется в метку времени UNIX, т. е. 1469848425 (различные серверные языки программирования напрямую обрабатывают это с помощью доступных функций времени). Затем метка времени преобразуется в шестнадцатеричную строку для дополнительного сокращения длины строки, то есть txTime = 1469848425 (десятичное) = 579C1B69 (шестнадцатеричное).

Как правило, txTime устанавливается на 24 часа позже текущего времени. Не рекомендуется устанавливать слишком короткий срок действия, чтобы избежать невозможности восстановления push-потока стримером в случае кратковременного сбоя сети во время трансляции.

- **Шаг 3: Генерация txSecret**
txSecret генерируется следующим образом: txSecret = **MD5 (KEY + stream_id + txTime)**. **KEY** здесь — это ключ шифрования, настроенный на Шаге 1. В данном примере stream_id равен 8888_test001, txTime — 579C1B69 (как рассчитано выше), а **MD5** — стандартный необратимый односторонний алгоритм хеширования.
- **Шаг 4: Формирование URL с защитой от хотлинкинга**
Объедините push (или playback) URL, txTime, указывающий время истечения срока действия URL, и txSecret, который может быть расшифрован и проверен только Tencent Cloud, чтобы сформировать безопасный URL с защитой от хотлинкинга.
