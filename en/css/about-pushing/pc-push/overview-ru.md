# Обзор

## Обзор PC CSS

![](https://staticintl.cloudcachetci.com/cms/backend-cms/c3256d5cd45011ee89c5525400170219.png)

Tencent Cloud PC CSS используется для отправки сжатых и закодированных изображений (таких как **прямые трансляции мероприятий**, **обучение**, **проекции** или **игры**) на **push URL** Tencent **Video Cloud** с помощью программного обеспечения для push (**OBS (рекомендуется)** или **XSplit**), установленного на ПК (**Windows/Mac**). При этом зрители могут видеть **изображения в реальном времени**, используя playback URL, соответствующий push URL.

## Процедура PC CSS

Вы можете легко реализовать PC CSS, выполнив следующие шаги:

- **Куда отправляются потоки**: Получите **push URL** и 3 playback URL из консоли Tencent Cloud CSS.
- **Что отправляется**: Настройте аудио/видео источники для push и параметры кодирования в стороннем программном обеспечении для push.
- Зрители могут смотреть CSS с помощью нашего RTMP DEMO, задав playback URL. Это позволяет **доставить контент зрителям**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/c324853cd45011ee9409525400c26da5.png)

## 1. Перед началом CSS

Активируйте облачный сервис CSS на Tencent Cloud. Если вы ещё не активировали сервис Tencent Cloud CSS, нажмите здесь, чтобы [подать заявку на подключение сервиса](https://console.tencentcloud.com/live).

![](https://staticintl.cloudcachetci.com/cms/backend-cms/c310a55ed45011eea2b0525400bb593a.png)

### 1.2 Генерация push URL

Если у вас нет **push URL**, вы можете сгенерировать **push URL** и 3 playback URL, перейдя в [Управление доступом > Доступ к CSS-коду > Конфигурация доступа](https://console.tencentcloud.com/live).

URL с доменным именем **livepush.myqcloud.com** является push-ссылкой:

![](https://staticintl.cloudcachetci.com/cms/backend-cms/c3102e50d45011ee89c5525400170219.png)

### 1.3 Выбор сети для CSS

- **Выбор сети**

| Тип сети | Доступность | Стабильность |
| --- | --- | --- |
| Проводная сеть | Низкая | Высокая |
| WIFI | Высокая | Низкая |

По возможности рекомендуется использовать проводную сеть, которая более стабильна, чем WIFI, и обеспечивает меньше помех. Для трансляций мероприятий рекомендуется использовать WIFI для удобства.

- **Тест пропускной способности исходящего канала**

Требования к пропускной способности исходящего канала зависят от качества видео и разрешения. Как правило, более высокое качество видео с более высоким разрешением означает более высокие требования к пропускной способности исходящего канала. Пропускная способность исходящего канала не должна быть менее 1 Мбит/с. Для проверки состояния пропускной способности исходящего канала можно воспользоваться [speedtest](http://www.speedtest.net/).

![](https://staticintl.cloudcachetci.com/cms/backend-cms/f5443b27d45011ee9409525400c26da5.png)

### 1.4 Установка программного обеспечения для push

- **Установка OBS**
Вы можете скачать установочный пакет на [официальном сайте OBS](https://obsproject.com/download) и установить его с настройками по умолчанию. OBS поддерживается в системах Windows/Mac/Linux. Убедитесь, что это Open Broadcaster Software. OBS также предоставляет OBS Studio, который не рассматривается в данном документе.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/0595e7fdd45111eea2b0525400bb593a.png)

- **Установка XSplit**

Вы можете скачать установочный пакет на официальном сайте XSplit и установить его с настройками по умолчанию.
XSplit — платное программное обеспечение. В качестве альтернативы можно использовать OBS (**бесплатно**). Для игровых CSS-трансляций XSplit имеет отдельный установочный пакет. Для неигровых CSS-трансляций рекомендуется использовать BroadCaster.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/c314dd66d45011eeb0d9525400461a83.png)

## 2. Настройка параметров программного обеспечения

### 2.1 Настройка push URL

Если вы получили следующий push URL:

> rtmp://3891.livepush.myqcloud.com/live/3891_test?bizid=3891&txSecret=xxx&txTime=58540F7F

Необходимо настроить две части отдельно:

> Передняя часть push URL **"rtmp://3891.livepush.myqcloud.com/live/"** называется **FMS URL**. Задняя часть push URL **"3891_test?bizid=3891&txSecret=xxx&txTime=58540F7F"** называется **stream code**.

- **Настройка push URL в OBS**

Нажмите **Settings** (Настройки), выберите **Broadcasting Settings** (Параметры трансляции), затем установите **Mode** (Режим) на **CSS Stream**, **Streaming Service** (Сервис потоковой передачи) на **Custom** (Пользовательский), **FMS URL** на первую часть push URL, а **Playback Path/Streaming Code** (Путь воспроизведения/Код потока) на вторую часть. Рекомендуется установить флажок **Automatic Reconnection** (Автоматическое переподключение), что означает автоматическое переподключение push при обнаружении OBS исключительной ситуации (например, отключения сети).

- **Настройка push URL в OBS Studio**

Нажмите **Settings** (Настройки) в правом нижнем углу страницы, выберите **Stream** (Поток), затем установите **Streaming Type** (Тип потока) на **Custom Streaming Media Server** (Пользовательский сервер потоковых медиа), **URL** на первую часть push URL, а **Stream Key** (Ключ потока) на вторую часть.

### 2.2 Настройка аудио- и видеоисточников

Аудио- и видеоисточники — это содержимое пакета данных, который будет отправлен. Существует три формата:

- Из устройств захвата видео, таких как камера или устройства видеозаписи.
- Из окон ПК или игровых источников.
- Из медиафайлов, хранящихся на ПК, таких как видеоизображения.
- **Настройка аудио- и видеоисточников OBS. Примечание**: **Нажмите правую кнопку мыши** в поле Source (Источник). Левая кнопка не применяется. Появится меню Add (Добавить), включающее **Acquire from Window** (Захват из окна), Acquire from Screen (Захват экрана), Image Source (Источник изображения), Slide Show (Слайд-шоу), Text Source (Текстовый источник), CLR Browser, **Video Capturing Device** (Устройство захвата видео), Game Source (Игровой источник) и т. д. Обычно используются Acquire from Window и Video Capturing Device. Другие источники настраиваются по-другому. Далее мы рассмотрим настройку **Video Capturing Device** (Устройство захвата видео).
- **Настройка аудио/видео источников OBS Studio**
Подробнее см. **Настройка аудио- и видеоисточников OBS**.

### 2.3 Настройка аудио- и видеоформатов

После настройки видеоисточника, несмотря на возможность получения аудио- и видеосигналов, исходные сигналы непригодны для передачи по сети из-за требований к полосе пропускания. Поэтому важно настроить параметры кодирования аудио и видео перед началом CSS.

| Параметр | Описание |
| --- | --- |
| **x264** | Кодировщик h264, наиболее широко используемый в отрасли, с более высоким коэффициентом сжатия видео при одинаковом качестве изображения. **Рекомендуется выбрать этот вариант**. |
| Nvidia NVENC | Выполняет кодирование с использованием ядра обработки видео, предназначенного для видеокарт nv. Требуется видеокарта Nvidia. |
| Quick Sync | Использует технологию Intel Quick Sync Video для поддержки аппаратного кодирования с высокой скоростью кодирования и качеством изображения. Однако имеет низкую совместимость и высокий битрейт. |
| **CBR** | Один из режимов управления битрейтом кодирования видео, называемый управлением постоянным битрейтом. Обеспечивает стабильный битрейт и больше подходит для сетевой передачи. **Рекомендуется выбрать этот вариант**. |
| **AAC** | Наиболее широко используемый формат кодирования аудио для прямых трансляций. **Рекомендуется выбрать этот вариант**. |

## 3. Проверка воспроизведения

### 3.1 Подтверждение playback URL

Если push URL (livepush):

> rtmp://3891.livepush.myqcloud.com/live/3891_test?bizid=3891&txSecret=xxx&txTime=58540F7F

то playback URL (liveplay):

| Протокол воспроизведения | Playback URL |
| --- | --- |
| FLV | `rtmp://3891.liveplay.myqcloud.com/live/3891_test` |
| RTMP | `http://3891.liveplay.myqcloud.com/live/3891_test.flv` |
| HLS(m3u8) | `http://3891.liveplay.myqcloud.com/live/3891_test.m3u8` |

### 3.2 Проверка воспроизведения через VLC

Нажмите здесь для [скачивания VLC](http://www.videolan.org/vlc/). Установите программу с настройками по умолчанию. Откройте программу, нажмите **Media Menu** (Меню Медиа), выберите **Open Network Stream** (Открыть сетевой поток), введите playback URL и нажмите **Play** (Воспроизвести).
